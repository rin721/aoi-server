# 服务器状态页面使用说明

服务器状态页面用于快速判断当前后端进程和主机资源是否健康。页面数据来自后端真实接口，不会展示前端伪造的 GPU、CI/CD 或服务进程明细。

## 进入页面先看哪里

1. 先看页面标题右侧的总体状态 Badge。它会把 CPU、内存、磁盘和 Go 堆内存中最需要关注的状态放在第一眼位置。
2. 再看 KPI 摘要。运行时长、CPU 平均、主机内存、磁盘水位、Go 堆和最近刷新时间用于快速扫视。
3. 如果看到异常提示，再进入对应资源卡片查看细项。

## 状态颜色含义

| 状态 | 含义 |
| --- | --- |
| 正常 | 指标处于健康范围。 |
| 关注 | 指标开始接近关注区间，可以继续观察。 |
| 预警 | 指标已经进入预警区间，应排查负载、容量或代码变化。 |
| 严重 | 指标进入高风险区间，应优先处理。 |
| 未知 | 当前接口没有返回可用数据。 |

## 每个区域怎么看

- CPU：顶部是平均使用率，下面是每个核心的负载条。核心很多时区域会局部滚动，不会把整个页面撑高。
- 主机内存：看总量、已使用、可用估算和使用率。页面会自动处理空值，不应出现 `undefined`、`null` 或 `NaN`。
- 磁盘：每个挂载点用进度条展示使用率，接近阈值的挂载点会优先靠前。
- Go 堆内存与 GC：用于判断当前 Go 进程的内存占用、对象数量、GC 暂停和下次 GC 目标。
- 构建信息：展示版本、模块、主包和 Go build settings；长值会换行或在区域内滚动。

## 刷新说明

刷新按钮会重新读取服务器状态。是否自动刷新、自动刷新间隔和手动刷新冷却时间由前端 Dashboard 配置控制，不要直接改页面代码里的数字。

## 新手提示

看到黄色或红色时，先看顶部异常摘要，再查看对应资源卡片。不要为了“让页面变绿”直接修改阈值；阈值应由维护者根据机器容量、业务峰值和运维策略在配置入口统一调整。
